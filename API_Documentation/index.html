- Get users filtered by name and connected -
URL: localhost:3000/friendRequest/ byName/userid/ connected/"first name" "last name"
Method: GET
Body: -----
    Parameters (explanation for the parameters in the URL):
  • - connected: connected = 1 return only friends, connected = 0 return all users according to the name filter
  • - "first name" "last name": "first name" (e.g "M") return users with first name starts with "M", Same for "last name". "first name" "last name" = " " return users without name filter.
    Answer from server:
  • - There are suitable users- JSON of the users configuration
  • - There aren't suitable users- "There are no suitable users to present"
- Get users by connected type -
URL: localhost:3000/friendRequest/ byType/userid/type/usersToPresent
Method: GET
Body: -----
    Parameters (explanation for the parameters in the URL):
  • - type: type = 1 user ask for his friends or requests only, type = 0 user ask for all users
  • - usersToPresent: list of users id to present. If the list include only "0" the user ask for all users
    Answer from server:
  • - There are suitable users in the DB- JSON of the users configuration with connection type according to the requesting user id
  • - There aren't suitable users- An empty table
